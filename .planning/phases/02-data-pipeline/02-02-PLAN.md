<task type="auto">
  <name>Task 1: Create normalization, reports, and build-data pipeline script</name>
  <files>src/lib/normalization.ts, src/lib/reports.ts, scripts/build-data.ts, package.json, .gitignore</files>
  <action>
    1. Create `src/lib/normalization.ts`:
       - normalizeForComparison(name: string): string
         - Trim, lowercase, collapse whitespace
         - Strip common suffixes: Inc., LLC, Corp., Holdings, Company, Co., Ltd., L.L.C., Corporation
         - Strip trailing commas and periods
         - Return cleaned string
       - generateNormalizationSuggestions(records: Dispensary[]): NormalizationSuggestion[]
         - Group records by normalizeForComparison(owner) AND normalizeForComparison(parentCompany)
         - Return groups where multiple distinct original spellings exist
         - Each suggestion: { normalizedKey, field ('owner' | 'parentCompany'), variants: string[], licenseNumbers: string[], recordCount }
         - Sort by recordCount descending (biggest groups first)

    2. Create `src/lib/reports.ts`:
       - generateQualityReport(records: Dispensary[], warnings: string[]): QualityReport
         - Count and list records where:
           - needsNarrative is true (empty Ownership Details)
           - researchInconclusive is true
           - phone is null/empty
           - address is null/empty
           - owner is null/empty
           - town is null/empty
         - Include the warnings from parser
         - Return: { totalRecords, issueCount, needsNarrative: { count, records: [{licenseNumber, tradeName}] }, researchInconclusive: { count, records }, missingFields: { phone: count, address: count, owner: count, town: count }, warnings }

    3. Create `scripts/build-data.ts`:
       - Import ExcelJS, fs, path from node:*
       - Import parseDispensarySheet, parseSummarySheet from src/lib/xlsx-parser (use relative path)
       - Import generateNormalizationSuggestions from src/lib/normalization
       - Import generateQualityReport from src/lib/reports
       - NOTE: This script runs outside Astro context via tsx, so imports from src/schemas/dispensary.ts that use 'astro/zod' need handling. Two options:
         - Option A: Have the schema file check if 'astro/zod' is available, fall back to 'zod'
         - Option B (preferred): Use tsconfig paths or tsx --tsconfig to alias 'astro/zod' to 'zod' at script runtime
         - The vitest.config.ts already aliases 'astro/zod' to 'zod'. Create a tsconfig.scripts.json that does the same for tsx, OR use the --import flag with a loader
         - Simplest: since tsx supports tsconfig paths, add a "paths" entry in tsconfig.json mapping "astro/zod" to node_modules/zod
       - Main flow:
         a. Resolve XLSX path: path.resolve(process.cwd(), 'MA_Dispensary_Ownership_Directory.xlsx')
         b. Check file exists, fail with clear message if not
         c. Read workbook with ExcelJS
         d. Parse Dispensary Directory sheet -> { records, errors, warnings }
         e. Parse Summary sheet -> stats
         f. Generate normalization suggestions from records
         g. Generate data quality report from records + warnings
         h. Create output directories: src/data/, data/reports/ (recursive mkdir)
         i. Write src/data/dispensaries.json -- array of Dispensary objects with "id" field set to licenseNumber (required by Astro file() loader)
         j. Write src/data/stats.json -- single stats object wrapped in array with id "summary"
         k. Write data/reports/data-quality-report.json
         l. Write data/reports/normalization-suggestions.json
         m. Console output:
            - "Processed N dispensary records"
            - "Data quality issues: N"
            - "  Needs narrative: N"
            - "  Research inconclusive: N"
            - "Normalization suggestions: N groups"
         n. If errors.length > 0: print each error, exit(1)
         o. If records.length === 0: print "No records found", exit(1)

    4. Update `package.json` scripts:
       ```json
       "build:data": "npx tsx scripts/build-data.ts",
       "build": "npm run build:data && astro build",
       "dev": "npm run build:data && astro dev"
       ```
       Install tsx as devDep: `npm install -D tsx`

    5. Update `.gitignore`:
       - Add `src/data/*.json` (generated files, rebuilt from XLSX)
       - Add `data/reports/` (generated reports)
       - Do NOT gitignore the scripts/ directory or src/schemas/

    6. Handle the 'astro/zod' import issue for scripts running outside Astro:
       - In tsconfig.json, add compilerOptions.paths: { "astro/zod": ["./node_modules/zod"] }
       - This allows tsx to resolve 'astro/zod' to 'zod' when running scripts/build-data.ts
       - Verify this does not break Astro's own resolution (Astro ignores tsconfig paths for its internal modules)
