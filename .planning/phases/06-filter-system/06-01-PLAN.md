  <files>src/lib/filter-logic.ts, tests/filter.test.ts</files>
  <behavior>
    - intersectSets: null + null = null; null + Set = Set; Set + null = Set; Set + Set = intersection
    - computeTagSlugs: empty activeTags = null; one tag = slugs with that tag; two tags = union (OR)
    - computeCountySlugs: empty county = null; "suffolk" = only suffolk slugs
    - computeVisibleSlugs: composes search + tags + county via intersection
    - computeTagCounts: given a base visible set, counts how many dispensaries per tag would match
    - computeCountyCounts: given a base visible set, counts how many dispensaries per county would match
  </behavior>
  <action>
    Create `src/lib/filter-logic.ts` with pure functions that operate on simple data structures (not DOM).
    These functions accept maps/sets of slug->tags and slug->county built from DOM at init time.

    Functions to implement:
    1. `intersectSets(...sets: (Set<string> | null)[]): Set<string> | null` — intersect non-null sets, null means "no constraint"
    2. `computeTagSlugs(activeTags: Set<string>, slugToTags: Map<string, string[]>): Set<string> | null` — OR union of slugs matching any active tag; null if no tags active
    3. `computeCountySlugs(activeCounty: string, slugToCounty: Map<string, string>): Set<string> | null` — slugs in county; null if no county selected
    4. `computeVisibleSlugs(searchSlugs: Set<string> | null, tagSlugs: Set<string> | null, countySlugs: Set<string> | null): Set<string> | null` — intersect all dimensions
    5. `computeTagCounts(slugToTags: Map<string, string[]>, baseSlugs: Set<string> | null, tags: string[]): Map<string, number>` — for each tag, count slugs that have it within baseSlugs
    6. `computeCountyCounts(slugToCounty: Map<string, string>, baseSlugs: Set<string> | null, counties: string[]): Map<string, number>` — for each county, count slugs within baseSlugs

    Write tests FIRST (RED), then implement (GREEN). Test with realistic data shapes (3-5 dispensary stubs).
    Use the existing vitest setup. Import from `src/lib/filter-logic.ts` using the project's path alias pattern.
  </action>
