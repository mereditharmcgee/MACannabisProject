    **Client-side `<script>` tag** (at bottom of file, Vite-processed):
    ```
    import Fuse from 'fuse.js';
    ```

    Implement the following logic:

    1. **State:** `let searchMatchSlugs = null;` (null = show all, Set = show only matching). This architecture supports Phase 6 filter composition.

    2. **Debounce utility:** 4-line debounce function, 150ms delay.

    3. **initSearch():** async function that:
       - Fetches `/data/search-data.json` and `/data/search-index.json` in parallel
       - Creates Fuse instance with parseIndex and the same config as build script (threshold: 0.3, ignoreLocation: true, minMatchCharLength: 1, shouldSort: true, keys with weights)
       - Checks URL for `?q=` parameter and executes queued search if present
       - Called on DOMContentLoaded

    4. **performSearch(query):** function that:
       - If query is empty/whitespace: set `searchMatchSlugs = null`, call `applyVisibility()`, return
       - Calls `fuse.search(query)`
       - Sets `searchMatchSlugs = new Set(results.map(r => r.item.slug))`
       - Calls `applyVisibility()`
       - Updates result count text
       - Updates URL with `history.replaceState`

    5. **applyVisibility():** function that:
       - Gets all `[data-slug]` elements
       - If `searchMatchSlugs === null`: show all cards, restore original alphabetical order
       - Otherwise: hide non-matching cards (`display: none`), show matching cards
       - For relevance ordering: use a DocumentFragment to reorder matching cards to top in result order, then append remaining hidden cards
       - Update `#result-count` text: "Showing X of 525 dispensaries" (X = visible count)
       - Toggle `#no-results` visibility: show if X === 0, hide otherwise; set query text in `#no-results-query`

    6. **Event listeners:**
       - `input` event on search-input: debounced `performSearch(input.value.trim())`
       - `click` on search-clear: clear input value, call `performSearch('')`, hide clear button
       - `input` event also toggles clear button visibility (hidden if empty, visible if text)
       - `popstate` event: read `e.state?.q`, set input value, call `performSearch`

    7. **URL state:**
       - `updateURL(query)`: uses `URLSearchParams` to set/delete `q` param, calls `history.replaceState({ q: query }, '', url)`
       - Called inside `performSearch` after debounce

    **Important implementation notes:**
    - Do NOT implement filter pills, dropdown, or any Phase 6 features. Only search.
    - Do NOT add text highlighting in cards. Cards look the same, just filtered.
    - Do NOT use fade animations. Cards appear/disappear immediately.
    - The `content-visibility: auto` on cards may interact with `display: none` toggling. If layout issues appear during testing, remove content-visibility from cards during active search (when searchMatchSlugs is not null).
    - Keep the card grid's existing class structure. Only add wrapper divs with data-slug.
